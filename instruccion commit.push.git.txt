Instrucciones Precisas para Commit y Push en el Repositorio Inandes.TECH

Este documento detalla los comandos exactos y el contexto necesario para realizar operaciones de Git (commit y push) en el repositorio 'Inandes.TECH'.

Contexto Importante:
- El directorio raíz del proyecto (donde se ejecuta el agente Gemini) es: C:\Users\rguti\
- El repositorio Git para este proyecto se encuentra en: C:\Users\rguti\Inandes.TECH\

Pasos para Realizar Commit y Push (solo archivos .py):

1.  **Añadir archivos al área de staging:**
    Este comando cambia temporalmente el directorio de trabajo a 'Inandes.TECH' y añade los archivos especificados al área de staging de Git.
    -   Para añadir todos los archivos Python:
        `cd Inandes.TECH && git add *.py`
    -   Para añadir un archivo específico (especialmente si el nombre contiene espacios, usa comillas):
        `cd Inandes.TECH && git add "nombre del archivo.txt"`
    -   Para añadir todos los cambios (archivos nuevos, modificados y eliminados):
        `cd Inandes.TECH && git add .`

2.  **Realizar el commit de los cambios:**
    Este comando cambia temporalmente el directorio de trabajo a 'Inandes.TECH' y crea un nuevo commit con los archivos staged.
    **IMPORTANTE:** Reemplaza "Tu mensaje de commit aquí" con un mensaje descriptivo de tus cambios.
    `cd Inandes.TECH && git commit -m "Tu mensaje de commit aquí"`

3.  **Subir los cambios al repositorio remoto (push):**
    Este comando cambia temporalmente el directorio de trabajo a 'Inandes.TECH' y sube los commits locales al repositorio remoto.
    `cd Inandes.TECH && git push`

Consideraciones Adicionales:
-   Estos comandos asumen que ya estás autenticado con tu repositorio remoto (por ejemplo, GitHub).
-   Si necesitas añadir otros tipos de archivos, ajusta el patrón `*.py` según sea necesario (ej. `git add .` para añadir todos los cambios).
-   Si encuentras errores relacionados con "not a git repository", asegúrate de que el directorio `C:\Users\rguti\Inandes.TECH` contenga la carpeta `.git`.
